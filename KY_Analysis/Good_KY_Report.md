# 「良いKY活動」徹底調査レポート

**作成日:** 2026-01-30  
**作成者:** Antigravity (AI Assistant)  
**目的:** 「良いKY活動」の本質を定義し、一人KYアプリ (Voice KY Assistant) の品質向上に役立てる。

---

## 1. 結論：「良いKY」の定義

調査の結果、「良いKY」とは**「情景が目に浮かぶほど具体的であること」**と定義できます。

| 要素 | 悪いKY（マンネリ・形骸化） | 良いKY（実効性が高い） |
| :--- | :--- | :--- |
| **表現** | 抽象的・スローガン的 | 具体的・記述的 |
| **構成** | 「〜に注意する」「〜を徹底する」 | 「(状況)なので、(現象)になり、(結果)になる」 |
| **視点** | いつもと同じ・コピー＆ペースト | その日の天候・体調・変化点が含まれている |

---

## 2. 危険予知の「3段構成」メソッド

質の高いヒヤリハット・危険予知文章を作成するための黄金法則は以下の3段構成です。

1. **状況 (Situation)**: 何をしている時、どんな状態で？（原因）
2. **現象 (Phenomenon)**: 何が起きる？（事故の型）
3. **結果 (Consequence)**: どうなる？（被害の程度）

### 📝 具体例比較

| カテゴリ | ❌ 悪い例（0点） | ⭕️ 良い例（100点） |
| :--- | :--- | :--- |
| **高所作業** | 足元に注意して作業する | 脚立の天板に乗って作業中、**バランスを崩して転落し、骨折する** |
| **重機作業** | 重機との接触に注意 | バックホーの旋回範囲内に立ち入り、**旋回部と接触して挟まれる** |
| **工具使用** | サンダーで手を切らないようにする | 軍手をしたまま回転部に手を近づけ、**手袋ごと巻き込まれて指を切断する** |
| **運搬作業** | 腰を痛めないようにする | 中腰の無理な姿勢で重量物を持ち上げ、**ぎっくり腰になる** |

> **💡 アプリへの応用:**
> AIのアドバイス機能で、「それは具体的にどういう状況で起きますか？」「怪我の程度は骨折ですか？擦り傷ですか？」と深掘り質問 (DLG-01) することで、ユーザーの入力を「良い例」に誘導できる。

## 3. 「一人KY」特有のポイント

チームで行うKYと異なり、一人KYには「他人の目がない」という最大のリスクがあります。これを補うためのテクニックです。

### ① 指差し呼称の徹底（Point & Call）

- **手法:** 対象を指差し、「〇〇よし！」と声を出す。
- **効果:** 脳の覚醒水準を上げ、誤操作・うっかりミスを **1/6** に減らす効果があると言われている。
- **アプリ応用:** 完了画面での「ヨシ！」ボタンや、音声操作時の発話推奨は理にかなっている。

### ② 「If（もしも）」思考

- **手法:** 「もし今、地震が起きたら？」「もしこのフックが外れたら？」と最悪のシナリオをシミュレーションする。
- **アプリ応用:** AIからの「逆張り質問 (DLG-04)」機能で、「もし突風が吹いたらどうしますか？」と問いかけるのが有効。

### ③ 場面転換時の「1分間KY」

- **手法:** 作業開始時だけでなく、「休憩明け」「場所移動時」「工具変更時」にリセットして行う。
- **アプリ応用:** アプリの起動ハードルを下げ、短時間（1分以内）で完結する設計が重要。

---

## 4. マンネリ化（形骸化）との戦い方

毎日同じ作業をしていると、思考停止（マンネリ）に陥ります。

- **一点集中テーマ:** 「今日は『足元』だけを見る」「今日は『頭上』だけを見る」と視点を絞る。
- **360度確認:** 自分の手元だけでなく、上下左右（他業者、落下物、埋設物）を確認する。
- **「昨日」との違い:** 「昨日は晴れていたが、今日は雨で足場が滑る」という「変化点」に着目する。

---

## 5. 本プロジェクトへの提言（Summary）

Voice KY Assistant の AI は、以下の役割を担うべきです。

1. **具体化のコーチ:** 「注意する」という言葉を禁止語にし、「どうなるか」を問いかける。
2. **視点の提供者:** 作業者が気づいていない「死角（他業者、天候変化）」を指摘する。
3. **マンネリブレイカー:** 毎日違う角度からの質問を投げかけ、思考を刺激する。

このレポートに基づき、Phase 2.4 (対話品質向上) のプロンプト設計を行うことを強く推奨します。

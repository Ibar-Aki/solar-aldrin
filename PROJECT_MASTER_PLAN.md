# AIファシリ型KYシステム (Voice KY Assistant) - Project Master Plan

**「話すだけで、質の高いKYが完了する」**
**「二度書きの無駄をなくし、現場の知恵をつなぐ」**

本ドキュメントは、プロジェクト全体の**ビジョン、ロードマップ、価値提案**を定義する最上位の企画書です。

---

## 1. エグゼクティブサマリー

### 背景と課題

建設現場における危険予知（KY）活動は、労働災害防止の要であるにも関わらず、以下の課題により**形骸化**が進んでいます。

1. **形骸化**: 作業員は「書くのが面倒」「考えるのが苦手」という二重の負担を抱え、毎回同じ内容を記入するルーチンワークと化している。
2. **二重入力のムダ**: 元請け提出用のKY用紙と、自社保管用のKY用紙が異なる場合、現場長は同じ内容を2回書く必要がある。
3. **知見の分断**: 個人のKY記録が紙で埋もれ、過去のヒヤリハットや他者の気付きが共有されない。

### ソリューション: Voice KY Assistant

**音声AIがKY活動をファシリテート**することで、以下の変革を実現します。

* **Input**: 「書く」負担をゼロにする（音声入力 & 画像取り込み）。
* **Process**: AIとの対話で「思考」を深め、潜在的な危険を洗い出す。
* **Output**: 監査に耐えうる記録を自動生成し、データを資産化する。

## 2. フェーズ別ロードマップ (Phased Strategy)

プロジェクトは以下の4フェーズで進化します。

| フェーズ | 呼称 | テーマ | 主な機能・価値 |
| :--- | :--- | :--- | :--- |
| **Phase 1** | **Legacy** | **実証実験 (MVP)** | ・音声対話によるKY入力の実現可能性検証<br>・(完了済 / 凍結中) |
| **Phase 2** | **Modern** | **個人の負荷ゼロ** | ・**現在開発中**<br>・React + Edgeによる高速な音声対話<br>・Web Speech APIによる入力負荷軽減<br>・PDF自動生成によるペーパーレス化 |
| **Phase 3** | **Expansion** | **視覚と知能の融合** | ・**【目玉】他社KY取り込み**: 元請け指定のKY用紙を撮影し、自社KYを自動補完<br>・**現場写真診断**: 現場写真をAI解析し、危険箇所を指摘 |
| **Phase 4** | **Organization** | **組織・データ活用** | ・**チームKY**: 職長と班員の会話からKYを生成<br>・**集合知の活用**: 過去のデータや他者の類似事例から、「この現場ではここが危ない」とリコメンド<br>・ダッシュボードによる傾向分析 |

## 3. 投資対効果 (ROI & Cost)

### コスト構造の革新

クラウドネイティブ（Serverless）技術と安価なAIモデルの採用により、圧倒的な低コスト運用を実現します。

| 項目 | 旧来想定 (VPS等) | **今回構成 (Cloudflare + OpenAI)** |
| :--- | :--- | :--- |
| **サーバー費用** | ¥3,000〜/月 | **¥0/月** (Workers無料枠内で十分動作) |
| **AI利用料** | ¥10〜20/回 (GPT-4) | **¥1.5/回 以下** (GPT-4o-mini)<br>※音声認識はブラウザ機能で無料 |
| **合計運用費** | **¥3,000〜/月** | **¥100〜/月** (個人ユースの場合)<br>※コーヒー1杯分で1ヶ月運用可能 |

### 期待される効果

* **工数削減**: 記入時間 10分 → 3分（▲70%）
* **質の向上**: 「ヨシ！」のみの記述 → 「足場板の結束確認ヨシ！」などの具体的記述へ
* **コンプライアンス**: 監査時の提出書類作成の手間ゼロ

---

## 4. プロジェクト方針

* **User First**: 現場作業員（特にITに不慣れな層）でも使える「迷わせないUI」を最優先する。
* **Speed**: レスポンスの遅延は思考の阻害要因となる。Edgeコンピューティングで極限までレイテンシを削る。
* **Safety**: 現場データは機密情報。画像データ等の取り扱いはセキュリティポリシーを厳守する。

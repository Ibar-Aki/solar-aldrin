# 個人利用のための実施計画

## 目的

このアプリを特定の現場や組織に限定せず、誰でも個人として安全に使える状態にするための実施計画をまとめます。

## 公開と配布

1. **公開先の明確化**
   - フロントエンドを一般公開できるホスティング（例: Cloudflare Pages）に配置し、誰でもアクセスできるURLを提供します。
   - PWAとしてインストールできるよう、HTTPSと適切なPWA設定を維持します。

2. **オンボーディングの整備**
   - 初回起動時に、使い方と注意点を日本語で短く案内します。
   - 音声入力が使えない環境に備え、テキスト入力の案内も行います。

3. **アカウント不要での利用**
   - 個人が気軽に試せるよう、基本機能はアカウントなしで使用できる設計を維持します。
   - 保存が必要な場合は、端末内保存やPDF出力で完結させる選択肢を用意します。

## セキュリティと個人情報

1. **個人情報の最小化**
   - 氏名や所属などの入力を必須にしない設計を維持します。
   - 入力内容は必要最小限に限定し、不要な個人情報を収集しない方針を明確にします。

2. **通信の保護**
   - すべての通信をHTTPSで行います。
   - 外部API（OpenAI、天候API）への送信内容を最小化し、送信前に注意喚起を表示します。

3. **データの保存方針**
   - 端末内保存やPDF出力が主な保存手段であることを明示します。
   - サーバー側にデータを保存する場合は、保存期間、削除方法、第三者提供の有無を明文化します。

## 信頼性と可用性

1. **オフライン対応の強化**
   - Service Workerのキャッシュを適切に更新し、オフラインでも基本機能が動作することを保証します。

2. **障害時の代替手段**
   - 音声認識が使えない場合に備え、テキスト入力への切り替えを分かりやすく提示します。

## コストと運用

1. **個人負担の見える化**
   - OpenAI APIなどの外部API利用に費用が発生する場合は、目安コストを明記します。

2. **利用上限と保護**
   - APIキーの不正利用を防ぐため、レート制限や使用量制限を設定します。

## 品質とサポート

1. **利用規約とプライバシーポリシー**
   - 一般公開する場合は、利用規約とプライバシーポリシーを用意します。
   - ユーザーが自分のデータについて把握できるよう、問い合わせ窓口を明記します。

2. **問い合わせ導線**
   - バグ報告や要望の送信先を案内します（例: GitHub Issues）。

## アクセシビリティ

1. **視認性と操作性**
   - 現場での利用を想定し、文字サイズやボタンの大きさを調整します。

2. **入力支援**
   - 音声認識の権限設定や、権限が拒否された場合の手順を説明します。

## 法令とガイドライン

1. **法令遵守**
   - 個人情報保護法など、該当する法令に沿った運用を行います。

2. **安全衛生の観点**
   - KY活動の趣旨を明記し、注意喚起として「最終的な安全確認は利用者が行う」旨を表示します。

## 実施順序の例

1. 公開URLの提供とPWA設定の確認
2. 利用規約・プライバシーポリシーの作成
3. 個人情報の最小化と保存方針の明文化
4. オフライン対応と音声入力の代替手段の改善
5. 問い合わせ導線の整備
6. コスト表示とAPI利用制限の設定

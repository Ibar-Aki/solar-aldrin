# Voice KY Assistant 企画・設計レビュー

対象: 企画書、要件定義書、詳細設計書
レビュー日: 2026-01-21
状態: **対応完了（MVP方針確定）**

---

## MVP方針（確定事項）

| 項目 | 方針 | 根拠 |
| ---- | ---- | ---- |
| 認証 | 不要 | 自己端末運用、1人利用 |
| 監査対応 | 署名欄はPDF印刷後に手書き | MVP簡略化 |
| 音声認識 | Web Speech API + テキスト入力フォールバック | 実装済み |
| オフライン | IndexedDB + 即時同期 | バッチ同期はv2 |
| 履歴閲覧 | 端末内のみ | 共有機能はv2 |

---

## 企画書・要件定義書レビュー

対象: `01_企画書.md`, `02_要件定義書.md`

### 指摘と対応

| 重要度 | 指摘 | 対応 |
| ------ | ---- | ---- |
| 高 | 認証不要とKY記録の実務要件が衝突 | MVP方針で確定（署名はPDF印刷後手書き） |
| 高 | 位置情報＋対話ログが個人識別に近い | 位置情報は天候取得のみ使用、ログ保存不要 |
| 高 | 音声認識の依存が強すぎる | テキスト入力フォールバック実装済み |
| 中 | オフライン対応の扱いが矛盾 | IndexedDB + 即時同期で対応 |
| 中 | "3タップ以内"がフローと矛盾 | 要件として削除検討 |
| 中 | アドバイス機能の遅延考慮なし | 対話完了後に非同期取得で対応 |
| 中 | 履歴閲覧のアクセス制御がない | 端末個人持ち前提でMVP許容 |
| 低 | コスト試算が価格変動を考慮していない | 参考値として記載 |
| 低 | MVP 2週間が楽観的 | 実装完了済み |

## 詳細設計書レビュー

対象: `04_詳細設計書.md`（v2）

### 採用理由

- 章立てが明確で、設計方針→構成→詳細→非機能の流れが一貫
- 状態管理・Service Worker・エラーコード・デプロイ手順まで含む
- 実装着手に必要な情報が網羅されている

### 指摘と対応

| 重要度 | 指摘 | 対応 |
| ------ | ---- | ---- |
| 高 | 監査必須項目がスキーマ欠落 | PDF署名欄で対応（手書き前提） |
| 高 | 履歴共有のアクセス制御なし | MVP方針で端末内限定 |
| 高 | オフライン同期設計が未定義 | 即時同期 + failed時再試行で実装 |
| 中 | /api/adviceがアーキテクチャ図に未反映 | 軽微、実装優先 |
| 中 | 履歴のページング設計なし | MVP規模で問題なし |
| 中 | 性能目標の実測根拠なし | パイロット運用で確認 |
| 低 | 命名の揺れ | MVP完了後にリファクタリング |
| 低 | キャッシュ戦略が機微データ未考慮 | 端末内限定で許容 |

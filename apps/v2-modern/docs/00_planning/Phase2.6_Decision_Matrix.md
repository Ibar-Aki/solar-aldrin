## 📍 Phase 2.6 の位置づけ

ロードマップへの新規追加フェーズ：

> **事後フィードバック・補強 (Feedforward & Supplement)**
> KYセッション完了後に、実施内容の「質」を評価してユーザーの成長を促すとともに、AIが足りない視点を補足して安全性を担保する。

**完了条件**:

- ユーザーが自分のKYの良い点・悪い点を認識できる
- ユーザーが見落とした重大な危険が記録として補完される

## 📝 Phase 2.6 機能候補

### ADV-01: KY品質フィードバック & アドバイス

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 今日のKYの良い点を見つけ出し、ポジティブな称賛と共に「さらなる成長のためのヒント」を提示する |
| **評価方針** | **減点法（採点）は行わない**。「具体性」「網羅性」などの観点から、良かったポイントを具体的に褒める。 |
| **目的** | ユーザーの自己効力感を高め、自発的なKYスキル向上を促す |
| **コスト** | 低（完了画面で評価用プロンプトを走らせるのみ） |
| **リスク** | なし（採点によるモチベーション低下リスクを排除） |
| **推奨案** | ✅ **採用**（「褒めて伸ばす」サイクルを作る） |

#### 🧠 補足検討：認知心理学的アプローチ

効果的なフィードバックのために、以下の心理学的手法を取り入れる。

1. **サンドイッチ法 (PNP法)**
    - ポジティブ(P) → 改善点(N) → ポジティブ(P) の順で伝える。
    - 例: 「『足場』に着目したのは素晴らしいです！(P) さらに『雨で滑る』という条件も加えると完璧ですね。(N) これで今日の安全はバッチリです！(P)」
2. **自己効力感 (Self-Efficacy) の向上**
    - 「あなたはリスクを見つける能力がある」と信じさせるメッセージを送る。
    - 成功体験（具体的にリスクを発見できたこと）を強調する。
3. **成長マインドセット (Growth Mindset)**
    - 「能力不足」ではなく「まだ伸びしろがある」と伝える。
    - 「惜しい！あと少しでプロレベルです」など、プロセスを評価する。

### ADV-02: 危険・対策のAI自動補足 (Supplement)

| 項目 | 内容 |
| :--- | :--- |
| **概要** | ユーザーが挙げた危険・対策に加え、AIが「この視点も大事です」と補足項目を提示する |
| **例** | 「💡 AI補足: 今日は風が強いので『飛散防止』も追加しておきました」 |
| **目的** | 人間の抜け漏れ（ヒューマンエラー）をAIがカバーする |
| **コスト** | 中（補足データの保存・表示フローの追加） |
| **リスク** | 「AIがやってくれるから適当でいいや」という依存を生む可能性 |
| **推奨案** | ✅ **採用**（安全担保の最後の砦として重要） |

---

### ADV-03: 類似災害事例の提示

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 今回の作業内容に近い、過去の実際の労働災害事例（厚労省DB等参照の体）を提示する |
| **例** | 「⚠️ 過去に似た作業で、こんな事故が起きています：『脚立の天板に乗り...』」 |
| **目的** | 「自分にも起きるかもしれない」という当事者意識（リアリティ）を持たせる |
| **コスト** | 中（事例データの準備またはRAG検索） |
| **リスク** | 怖がらせすぎて萎縮させる可能性 |
| **推奨案** | ⏸ **保留**（まずはADV-01/02で基礎を固める） |

---

### ADV-04: 行動目標のブラッシュアップ

| 項目 | 内容 |
| :--- | :--- |
| **概要** | ユーザーが決めた行動目標（指差し呼称項目）を、よりリズム良く覚えやすいフレーズにAIが添削提案する |
| **例** | 入力:「足元を確認してあわてない」→ 提案:「足元確認、あわてずヨシ！」 |
| **目的** | 現場で唱えやすい「キレのある」行動目標にする |
| **コスト** | 低（プロンプト調整のみ） |
| **リスク** | こだわりがあるユーザーには余計なお世話になる |
| **推奨案** | ✅ **採用**（「ヨシ！」の質を高める） |

# Phase 2.8 機能判定シート (Decision Matrix)

**目的**: Phase 2.8 (現場UX強化・スタート画面改善) で実装する機能の詳細を整理し、要否判定の材料とする。

**作成日**: 2026-01-30

---

## 📍 Phase 2.8 の位置づけ (Proposal)

ユーザー要望「UI/UXの強化（特にスタート画面）」に基づく新設フェーズ：

> **現場UX強化・状況入力 (Field UX+)**
> KY開始前の「基本情報入力」を強化し、ユーザーがスムーズに思考モードに入れるようにする。また、入力された情報をコンテキストとしてAIに渡すことで助言精度を高める。

**完了条件**:

- アプリ起動直後に「今日の日付」「工程」「体調」が直感的に入力/確認できる
- 入力された情報がKYセッションのコンテキスト（前提情報）として機能する

---

## 📝 Phase 2.8 機能候補

### UX-10: スタート画面での日付・天気表示

| 項目 | 内容 |
| :--- | :--- |
| **概要** | スタート画面の最上部に、今日の日付、曜日、(可能なら天気) を大きく表示する |
| **目的** | 毎朝のアプリ起動時に「今日」を意識させ、リズムを作る |
| **技術** | `date-fns` 等でローカル日時を表示。天気は `OpenWeatherMap` (C-01) と連動想定。 |
| **コスト** | 低 |
| **推奨案** | ✅ **採用**（基本機能として必須） |

---

### UX-11: 大工程の選択 (Process Selector)

| 項目 | 内容 |
| :--- | :--- |
| **概要** | KY開始前に、今日の作業工程を以下の6択からワンタップで選ばせる |
| **選択肢** | ①搬入・荷受け、②基礎土台・建地準備、③組み立て、④付帯設備設置・仕上げ、⑤引き渡し前確認、⓪フリー |
| **目的** | AIが「どの段階の作業か」を理解し、的外れな指摘（例: 仕上げ段階なのに基礎の話をする等）を防ぐ |
| **コスト** | 低（選択UIの実装 + プロンプト注入） |
| **推奨案** | ✅ **採用**（AI精度向上に直結） |

---

### UX-12: 体調チェック (Condition Check)

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 今日の体調を3段階（悪い・よい・すごくよい）のアイコン等で選択させる |
| **目的** | 作業者の自覚を促す。「悪い」選択時にAIが「無理しないでくださいね」と労うことで信頼関係を作る。 |
| **コスト** | 低（UI実装 + プロンプト分岐） |
| **推奨案** | ✅ **採用**（安全管理の基本） |

---

### UX-13: 無災害継続日数の表示 (Safety Streak) [提案]

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 「無災害 継続 〇〇日」というカウンターを表示する |
| **目的** | ゲーム感覚（ストリーク）で安全意識を持続させる |
| **コスト** | 中（ローカルDBでのカウントロジック実装） |
| **推奨案** | ⏸ **保留/提案**（まずはUX-10〜12を優先） |

# Phase 2.4 機能判定シート (Decision Matrix)

**目的**: Phase 2.4 (対話品質向上) で実装する機能の詳細を整理し、要否判定の材料とする。

**作成日**: 2026-01-30

---

## 📍 Phase 2.4 の位置づけ

ロードマップ (`03_Phase2ロードマップ`) における Phase 2.4 の定義：

> **対話品質の向上**
> 5W1H深掘り、段階的負荷、選択式深掘り、4Rフロー、ペルソナ調整、短い振り返りにより対話の質を高める。

**完了条件**:

- 抽象回答が具体化されること
- 重要危険が毎回1つ選ばれること
- 自己決定感の簡易評価が維持/向上

---

## 💬 Phase 2.4 機能候補

### DLG-01: 5W1H深掘り + 段階的負荷

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 「いつ」「どこで」「誰が」「何を」「なぜ」「どのように」を順次質問し、具体性を高める |
| **目的** | 「転落に注意」のような抽象回答を「足場3段目の踊り場で、雨天時に滑って転落」まで具体化 |
| **実装案** | プロンプト設計 + AIの応答パターン調整 |
| **コスト** | 低〜中（プロンプトエンジニアリング） |
| **リスク** | 質問が多すぎると作業者が面倒に感じる |
| **推奨案** | ✅ **採用**（対話品質の核心機能） |

---

### DLG-02: 選択式深掘りUI

| 項目 | 内容 |
| :--- | :--- |
| **概要** | AIが選択肢を提示し、タップで回答できるようにする |
| **例** | 「危険度は？」→ [1] [2] [3] [4] [5] のタップボタン |
| **目的** | 入力の手間を減らし、回答の標準化を促進する |
| **コスト** | 中（UI実装 + AI応答形式の調整） |
| **リスク** | 選択肢に収まらない回答の扱いが課題 |
| **推奨案** | ✅ **採用**（UX-03 ハイブリッド入力との統合を検討） |

---

### DLG-03: 4Rフロー + 重要危険の選択UI

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 4R（危険発見→理由分析→対策決定→レビュー）の流れを明確化し、最も重要な危険を1つ選ばせる |
| **目的** | KY活動の本質である「重点リスクへの集中」を促す |
| **実装案** | セッション終了前に「最も注意すべき危険はどれ？」の選択UIを追加 |
| **コスト** | 低（UI追加 + プロンプト調整） |
| **リスク** | 追加ステップが負担になる可能性 |
| **推奨案** | ✅ **採用**（KY品質の差別化に直結） |

---

### DLG-04: テンプレローテ + 逆張り質問

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 質問のバリエーションを増やし、予想外の角度から問いかけることでマンネリを防ぐ |
| **例** | 「今日は絶対にやらないことを1つ決めてください」（逆張り） |
| **目的** | 「いつも同じ質問」による思考停止を防ぐ |
| **コスト** | 低（プロンプトのバリエーション追加） |
| **リスク** | 逆張りが理解されず混乱を招く可能性 |
| **推奨案** | ✅ **採用**（低コストでマンネリ抑止） |

---

### DLG-05: ペルソナ「新人記録係」

| 項目 | 内容 |
| :--- | :--- |
| **概要** | AIのペルソナを「ベテランの安全管理者」から「新人の記録係」に変更する |
| **目的** | 「教えられる」感覚を減らし、「自分が主体」という意識を高める |
| **実装案** | システムプロンプトのペルソナ設定を変更 |
| **コスト** | 極低（プロンプト変更のみ） |
| **リスク** | 信頼性が下がると感じるユーザーがいる可能性 |
| **推奨案** | ⏸ **試験的採用**（A/Bテストで効果検証） |

---

### DLG-06: 短い振り返り（メタ認知）

| 項目 | 内容 |
| :--- | :--- |
| **概要** | セッション終了時に「今日気づいたこと」を一言で振り返らせる |
| **目的** | 内省を定着させ、KY品質を底上げする |
| **実装案** | 完了画面に自由記述欄を追加（任意） |
| **コスト** | 極低（UI追加のみ） |
| **リスク** | 入力が面倒でスキップされる可能性 |
| **推奨案** | ✅ **採用**（任意入力なら負担は最小） |

- **DLG-06**: 不採用確定。

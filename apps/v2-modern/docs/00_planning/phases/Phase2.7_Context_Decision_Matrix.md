# Phase 2.7 機能判定シート (Decision Matrix)

**目的**: Phase 2.7 (コンテキスト注入/ナレッジ循環) で実装する機能の詳細を整理し、要否判定の材料とする。

**作成日**: 2026-01-30

---

## 📍 Phase 2.7 の位置づけ (Renumbered)

> **コンテキスト注入/ナレッジ循環**
> Phase 2.3 で蓄積された履歴データ（ヒヤリハット等）を参照し、AIが「毎回同じ助言」をしないようにする。

**完了条件**:

- 「毎回同じ助言」を抑制できる
- 過去の危険が再利用される

---

## 🔄 Phase 2.7 機能候補

### CTX-01: 過去ヒヤリハット参照

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 過去のKYセッションで記録されたヒヤリハット情報を参照し、AIが助言に活用する |
| **前提** | Phase 2.3 (History) の実装完了 |
| **推奨案** | ✅ **採用**（ナレッジ循環の核心機能） |

---

### CTX-02: 昨日の指摘参照

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 直近（昨日〜3日以内）に同じ現場で指摘された危険を優先的に表示する |
| **推奨案** | ✅ **採用**（CTX-01 と組み合わせて効果大） |

---

### CTX-03: 簡易RAG（履歴検索 + プロンプト注入）

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 過去のKY記録を検索し、関連する危険・対策をAIプロンプトに注入する |
| **推奨案** | ⏸ **保留/試験的**（まずはCTX-01/02で効果検証してから） |

---

### CTX-04: 天候・曜日の文脈強化

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 天候（雨・強風等）や曜日（週明け・金曜等）に応じた危険を強調する |
| **推奨案** | ✅ **採用**（低コストで効果が見込める） |

---

### CTX-05: 現場固有ナレッジ登録

| 項目 | 内容 |
| :--- | :--- |
| **概要** | 現場ごとの固有リスク（例：「3階の足場が老朽化」）を事前登録し、参照する |
| **推奨案** | ⏸ **保留**（Phase 3.0 のナレッジプラットフォームに統合を検討） |

# Voice KY Assistant — 機能概要

**AI × 音声で、KYを「やらされる作業」から「自分で気づく習慣」へ。**

更新日: 2026-02-21
バージョン: v2-modern（Phase 2.10 完了）

---

## このアプリでKYがどう変わるか

Voice KY Assistantは、建設現場の危険予知（KY）活動を、AIとの対話で進められるWebアプリです。
紙とペンだけの従来KYでは見落としがちだった危険を、AIパートナーが質問で引き出し、
対策の抜け漏れを補足し、「褒めて伸ばす」フィードバックで安全意識を定着させます。

**特長:**
スマートフォンのブラウザだけで動作。アプリのインストールは不要です。
音声入力にも対応しているため、手袋をしたままでも操作できます。

## 実装済み機能一覧

### 1. AIパートナーとのKY対話

AIが「KYパートナー」として、作業者と一緒に危険を洗い出します。
一方的に聞くだけではなく、対話を通じて作業者自身が気づける仕組みです。

| 機能 | 内容 | KYへの効果 |
|:-----|:-----|:-----------|
| 段階的な対話進行 | 作業内容 → 危険 → 要因 → 対策の順に、AIが1つずつ質問 | 考えが整理され、抜け漏れが減る |
| 対策の3分類 | 「設備・環境」「配置・行動」「保護具」の3カテゴリで対策を整理 | 偏りのない対策検討ができる |
| 危険度の選択 | 1〜5段階で危険度を評価、ボタンで即入力 | 重点管理すべきリスクが明確になる |
| 複数危険の処理 | 1回のセッションで最大2件の危険を順番に検討 | 主要リスクを網羅できる |
| 行動目標の決定 | 「今日一番意識する行動」を最後に宣言 | 安全行動が具体化・習慣化する |
| コンテキスト注入 | 直近の記録で繰り返し出ている危険を自動参照 | 「またか」を「だからこそ」に変える |

---

### 2. KYボード（リアルタイム可視化）

会話の進行に合わせて、画面上の「KYボード」がリアルタイムで埋まっていきます。

| 表示項目 | 内容 |
|:---------|:-----|
| 何をするとき | 作業内容（例：脚立上で天井配線を固定する時） |
| 何が原因で | 危険の要因（例：設置角度が不適切で足元が滑りやすい） |
| どうなる | 想定される災害（例：バランスを崩して墜落し、頭部を負傷） |
| 対策 | カテゴリ別の対策一覧 |
| 危険度 | 4段階のセグメントバーで視覚表示 |

> 会話しながら「今どこまで進んだか」が一目でわかるため、記入漏れを防げます。

---

### 3. 事後フィードバック（AIによる品質向上）

セッション完了後、AIがKY活動全体を分析し、4つの観点でフィードバックを返します。

| フィードバック項目 | 内容 | 狙い |
|:-------------------|:-----|:-----|
| 良かった点（褒め） | 具体的に何が良かったかを称賛 | 自己効力感を高め、継続意欲を維持 |
| 次回へのヒント | 「いつ・どこで・何を確認するか」を1つだけ提示 | 次のKYが確実に良くなる |
| 補足リスク | ユーザーが挙げなかった危険を最大2件追加 | 見落としの最終防止 |
| 行動目標の磨き込み | 宣言した目標を唱えやすいフレーズに整形 | 指差し呼称で使える形にする |

> 「サンドイッチ法（褒め → 改善 → 褒め）」で、否定感なく改善を促します。

---

### 4. 音声機能

現場での使いやすさを重視し、3つの音声機能を搭載しています。

| 機能 | 内容 | 利用場面 |
|:-----|:-----|:---------|
| 音声入力（マイク） | 話した内容をテキストに自動変換 | 手が塞がっているとき |
| AI応答の読み上げ（TTS） | AIのメッセージを音声で再生 | 画面を見られないとき |
| 完全音声会話モード | 自動読み上げ＋自動音声認識＋自動送信 | ハンズフリー運用 |

> 完全音声会話モードでは、TTS再生中にマイクを自動停止し、
> 再生終了後に音声認識を再開します。エコーや二重認識を防ぐ設計です。

---

### 5. 記録・履歴・エクスポート

KY活動の記録を端末内に保存し、いつでも振り返れます。

| 機能 | 内容 |
|:-----|:-----|
| PDF記録ダウンロード | KY活動の内容をPDFとして保存・提出 |
| 履歴一覧・詳細表示 | 過去のKY記録を一覧で確認し、詳細を開ける |
| 履歴からの引用スタート | 過去の記録から基本情報をコピーして新規KYを開始 |
| CSV / JSONエクスポート | 管理者への提出や分析用にデータを出力 |
| 自動データ整理 | 90日または100件を超えると古い記録を自動削除 |

> 履歴はブラウザ内（IndexedDB）に保存されるため、サーバーに個人情報は送信しません。

### 6. 体験を支える演出・UI

| 機能 | 内容 |
|:-----|:-----|
| 完了演出（ファンファーレ） | セッション完了時に達成感を演出 |
| 天候・体調の入力UI | ワンタップで選べるセレクタ |
| 直近リスクバッジ | 繰り返している危険を視覚的に注意喚起 |
| スケルトンUI | AI応答待ちの間、読み込み中UIを表示 |

---

### 7. セキュリティ・信頼性

安心して業務利用できるよう、多層的なセキュリティ対策を施しています。

| 対策 | 内容 |
|:-----|:-----|
| レート制限 | APIの過剰利用・不正アクセスを自動ブロック |
| Origin制限 | 許可されたドメインからのみアクセス可能 |
| 入力バリデーション | 不正な入力値をサーバー側で検証・遮断 |
| PIIマスキング | ログにメールアドレス・電話番号などが記録されない |
| JSON回復・リトライ | AI応答が破損しても自動修復し、セッションを継続 |
| エラー監視（Sentry） | エラーを自動検知し、品質改善に活用 |
| パフォーマンス計測 | Web Vitals（読み込み速度等）を常時計測 |
| マルチAIプロバイダ | 複数のAIサービスに対応し、障害時の切り替えが可能 |

## 動作環境

| 項目 | 内容 |
|:-----|:-----|
| 対応ブラウザ | Chrome / Safari / Edge（最新版推奨） |
| 対応端末 | スマートフォン / タブレット / PC |
| ネットワーク | AI対話・フィードバックにはインターネット接続が必要 |
| 音声機能 | ブラウザの音声認識・音声合成に対応した端末が必要 |

> 履歴・PDF出力はオフラインでも利用できます。
> 音声認識は端末・ブラウザの対応状況に依存します（非対応端末ではテキスト入力で代替）。
